# Notice 页面 UI 打磨设计

> **目标**：将 Notice 通知公告页面从"功能可用"升级为"视觉完善"，对标飞书管理后台的成熟度，使用 shadcn/ui 组件重新组合。
>
> **范围**：仅改 L5 Notice 页面层（`web-admin/src/features/notice/`），不改 L3 组件。

---

## 1. 设计决策

| 维度 | 决策 | 理由 |
|------|------|------|
| 列表页布局 | 筛选面板式 | 筛选条件在表格上方独立一行，表格+分页包在 Card 内，适合复杂筛选场景 |
| 编辑表单 | 全屏 Dialog | 左侧表单 + 右侧设置面板，富文本编辑需要宽屏空间 |
| 详情页 | 独立路由页 | `/notices/:id`，返回按钮回列表 |
| 行操作 | 主操作外露 + ⋯ 收纳 | 常用操作（详情/编辑）直接显示，次要操作（发布/撤回/复制/删除）收到 DropdownMenu |

## 2. 列表页（`notice-list-page.tsx`）

### 2.1 页面结构

```
┌─────────────────────────────────────────────────┐
│ 面包屑：系统管理 > 通知公告                          │
├─────────────────────────────────────────────────┤
│ 标题：通知公告              [导出] [+ 新建公告]      │
├─────────────────────────────────────────────────┤
│ 🔍 搜索公告标题...  状态[全部▾] 优先级[全部▾]  重置   │
├─────────────────────────────────────────────────┤
│ ┌─── Card ────────────────────────────────────┐ │
│ │ ☐  标题         状态    生效时间  已读率  操作 │ │
│ │ ☐  📌系统维护..  已发布  04-15   ██ 60%  详情⋯│ │
│ │ ☐  新功能上线..  已发布  04-14   ███90%  详情⋯│ │
│ │ ☐  周末值班..    草稿    —       —       编辑⋯│ │
│ ├─────────────────────────────────────────────┤ │
│ │ 共 3 条                    每页[10▾] ‹ 1/1 › │ │
│ └─────────────────────────────────────────────┘ │
│                                                   │
│ ┌─── NxBar（选中时出现）────────────────────────┐ │
│ │ 已选 2 条  [清除]          [批量发布] [批量删除]│ │
│ └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

### 2.2 改动要点

**页面 Header 区**：
- 新增面包屑（Breadcrumb 组件，路径：系统管理 > 通知公告）
- 标题 + 副标题 + 右侧操作按钮（导出 + 新建）作为 flex 布局
- 背景色使用页面默认背景（不是白色 Card）

**筛选区**（替换当前 NxFilter）：
- 不再使用 NxFilter 组件，改为直接用 L2 组件组合横排筛选条
- 搜索框（Input + 搜索图标前缀）
- 状态下拉（Select，选项：全部/草稿/已发布/已撤回）
- 优先级下拉（Select，选项：全部/普通/紧急/特急）
- 重置按钮（文字按钮，清除所有筛选）
- **即时筛选**：onChange 直接触发查询（不再需要"查询"按钮），搜索框用 debounce 300ms
- 状态变化同步到 URL search params

**表格区**（Card 包裹）：
- 整个表格 + 分页用 `Card` 组件包裹
- 表头背景色：`bg-muted/50`，文字 `text-muted-foreground`
- 行间距加大：padding `py-3 px-4`
- 置顶标记：📌 图标（用 lucide `Pin` 组件）
- 状态列：`Badge` 组件，颜色映射（草稿=secondary, 已发布=success, 已撤回=destructive）
- 已读率列：进度条（div 模拟）+ 百分比文字
- 操作列：
  - 草稿状态：`编辑` 文字按钮 + `⋯` DropdownMenu（发布/复制/删除）
  - 已发布状态：`详情` 文字按钮 + `⋯` DropdownMenu（撤回/复制）
  - 已撤回状态：`详情` 文字按钮 + `⋯` DropdownMenu（复制/删除）
- 行 hover：`hover:bg-muted/50` 过渡效果
- 未读行：左侧 2px 蓝色边框指示

**分页区**（Card 内底部）：
- 左侧：选中统计（"已选 N 条，共 M 条"）
- 右侧：每页条数选择（Select, 10/20/50）+ 页码信息 + 上一页/下一页按钮
- 分隔线 `border-t` 与表格分开

### 2.3 不再使用的 L3 组件

- ~~NxFilter~~：改为直接组合 L2 Input/Select，实现即时筛选
- NxTable：**继续使用**，但需要调整 className 传入样式覆盖
- NxBar：**继续使用**，固定底栏批量操作

## 3. 编辑弹窗（替换 `notice-drawer.tsx`）

### 3.1 结构

```
┌─────────── Dialog（全屏，max-w-5xl）─────────────┐
│ ← 返回  新建公告              [存为草稿] [保存并发布] │
├────────────────────────────────┬──────────────────┤
│                                │                    │
│  标题 *                        │  发布设置            │
│  ┌─────────────────────────┐  │                    │
│  │ 输入公告标题              │  │  优先级              │
│  └─────────────────────────┘  │  ┌──────────────┐  │
│                                │  │ 普通 ▾        │  │
│  公告内容                      │  └──────────────┘  │
│  ┌─────────────────────────┐  │                    │
│  │ B I U | H1 H2 | 📷 📎   │  │  ☐ 置顶           │
│  │                          │  │                    │
│  │ 在此编辑公告内容...       │  │  生效时间           │
│  │                          │  │  ┌──────────────┐  │
│  │                          │  │  │ 选择日期       │  │
│  └─────────────────────────┘  │  └──────────────┘  │
│                                │                    │
│  附件                          │  失效时间           │
│  ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐  │  ┌──────────────┐  │
│  │ 📎 拖拽文件或点击上传    │  │  │ 选择日期       │  │
│  └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘  │  └──────────────┘  │
│                                │                    │
│                                │  发布目标           │
│                                │  ┌──────────────┐  │
│                                │  │ 全部用户 ▾     │  │
│                                │  └──────────────┘  │
└────────────────────────────────┴──────────────────┘
```

### 3.2 改动要点

**组件替换**：
- 从 NxDrawer（右侧抽屉）改为 Dialog（全屏弹窗，`max-w-5xl`）
- 使用 `DialogContent` + `DialogHeader` + `DialogTitle`

**布局**：
- 双栏布局：左侧主表单区（flex-1）+ 右侧设置面板（w-64，bg-muted/30 + rounded-lg + padding）
- 右侧面板包含：优先级 / 置顶开关 / 生效时间 / 失效时间 / 发布目标
- 左侧包含：标题输入 / 富文本编辑器（TipTap）/ 附件上传区

**Header**：
- 左侧：返回按钮 + 标题（"新建公告" 或 "编辑公告"）
- 右侧：操作按钮（存为草稿 + 保存 或 保存并发布）
- 编辑模式下根据当前状态显示不同按钮组合

**表单校验**：
- 继续使用 React Hook Form + Zod
- 脏检查：关闭弹窗时如果 isDirty 弹 AlertDialog 确认

**附件区**：
- 虚线边框 dashed drop zone
- 支持拖拽（HTML5 drag & drop）+ 点击上传
- 已上传文件列表展示（文件名 + 大小 + 删除按钮）

## 5. 技术方案

### 5.1 改什么

| 文件 | 改动 |
|------|------|
| `notice-list-page.tsx` | **重写**：去掉 NxFilter，用 L2 组件组合筛选区 + Card 包裹表格 + 新分页 + 行操作 DropdownMenu |
| `notice-drawer.tsx` → `notice-dialog.tsx` | **重写**：NxDrawer → Dialog 全屏弹窗，双栏布局 |
| `notice-detail-page.tsx` | **重构**：加 Card 包裹 + Header 优化 + 元信息区 |
| `notice-status-badge.tsx` | **微调**：确保用 Badge variant 映射正确颜色 |
| `batch-confirm-dialog.tsx` | **不动** |
| `tiptap-field.tsx` | **不动** |
| `file-upload-field.tsx` | **微调**：加 drag & drop 视觉区 |
| `target-selector.tsx` | **不动** |
| `recipients-tab.tsx` | **不动** |
| `notification-log-tab.tsx` | **不动** |

### 5.2 不改什么

- **L3 组件**（NxTable/NxBar/NxForm 等）：保持不变
- **路由文件**：`/notices/index.tsx` 和 `/notices/$id.tsx` 不动
- **后端**：零改动
- **i18n**：需要补充新增的 key（面包屑、筛选 placeholder 等）
- **MSW mock**：可能需要补充 mock 数据

### 5.3 新增依赖

无。所有需要的组件（Card, Badge, Dialog, DropdownMenu, Select, Input, Breadcrumb, Tabs, Separator）都已在 L2 导出。
