# 0413 后端第二轮优化 + 前后端联查 + 愿景对齐

## 后端 specs 第二轮优化（4 路 Codex 对抗性审查）

- 问题：发现 6 Critical + 7 High + 12 Major
- 过程：逐个问洋哥决策，12 项决策全部确认后 4 路 agent 并行落地
- 核心决策：
  - C6: Sa-Token JWT Mixed → Simple 模式（全走 Redis，功能拉满）
  - C4: NO_DIRECT_TIME_API 删除 ArchUnit 规则，改为工具类+文档引导
  - C5: SENSITIVE_RECORDS 规则删除，降级为文档约定
  - H3: 基础设施健康度策略（Redis/PG fail-fast + Actuator + 告警）
  - H4+H6: DataScopeVisitListener 只在主表注入（大幅降低复杂度）
  - M5: Access 30min + Refresh 30天 + Rotation 入侵检测
  - D3: 渐进延迟+CAPTCHA 替代硬锁定，新增 infra-captcha 子模块
  - D4: Actuator 默认值从宽松改为安全
  - 文件存储: MinIO → 本地优先 + 阿里云 OSS
- 结论：后端 112 + 前端 138 = 250 项 verify 全绿
- 耗时：约 4 小时

## 数据建模深入讨论

- 数据权限 owner_dept_id 方案：冗余字段，创建时写入不随调岗更新
- 用户/部门生命周期：ACTIVE/DISABLED/DELETED 状态机 + Partial Unique Index
- 快照 vs 关联：凭证类存快照（订单行项价格/审批人/操作日志），展示类用 JOIN/缓存
- 输出独立文档：《快照数据vs关联数据-业务系统数据建模决策指南》
