## 完成内容

### jOOQ codegen 方案重构
- **问题**：昨晚落地的 gmaven-plus 一体化方案把 Flyway 调用塞进 Groovy 脚本，导致 8 个 plugin deps
- **调研**：按决策四步协议，从官方文档出发调研了 6 种方案（DDLDatabase / 三插件 / 专用插件 / jdbc:tc: URL 等）
- **结论**：切换到 jOOQ 官方推荐的三插件模式（groovy-maven-plugin → flyway-maven-plugin → jooq-codegen-maven），plugin deps 从 8 个降到 2 个
- **关键知识**：Maven 每个插件独立 ClassLoader，static 字段隔离，`jdbc:tc:` URL 无法跨插件共享容器

## 决策记录

| 决策 | 结论 | 理由 |
|------|------|------|
| jOOQ codegen 方案 | 三插件模式 | jOOQ 官方推荐 + 职责分离 + 依赖隔离 |
| DDLDatabase | 排除 | H2 模拟不支持 PG 特有语法（TIMESTAMPTZ、CREATE INDEX CONCURRENTLY 等） |
| Docker 端口 | 加 1 前缀 | 避免与本机原生服务冲突，对所有开发者统一 |
| 合并策略 | --no-ff 保留全部 commit | M1 每个 commit 是独立逻辑单元，git blame 有用 |
| tenantId 类型 | Long（包装类型） | 容错优先，内部用工具方法做 null 校验 |
| Branch protection | 暂不开启 | GitHub 免费版私有仓库不支持，等 M7 开源 |
