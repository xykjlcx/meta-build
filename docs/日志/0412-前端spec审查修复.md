# 0412 前端 spec 五维审查 + P0/P1 修复 + Codex review

## 五维交叉审查
- 问题：前端 spec 13 个文件 ~11700 行由 6 个 agent 并行编写，需要系统性审查设计健全性
- 过程：dispatch 5 个审查 agent（内部一致性 / 前后端契约 / 技术可落地 / 架构健全性 / 硬约束可执行性）并行审查全部文件
- 结论：去重后 7 P0 + 15 P1 + 8 P2
- 耗时：审查 ~20 分钟（5 agent 并行）

## P0 决策（7 个）
- 问题：L4↔api-sdk 循环依赖 / Tailwind 3.4 不支持 oklch opacity / i18next 插件不存在 / SPA 刷新竞态 / token 存储混乱 / 后端 deleted 残留 / Biome 规则不存在
- 过程：逐项和洋哥讨论，应用"决策四步协议"（是不是→是什么→好坏→怎么做）。P0-3 和 P0-5 通过"是不是"步骤发现问题本身消失（不需要 i18next 工具 / 不需要 zustand）
- 结论：L4 放行 api-sdk / Tailwind v4 / module augmentation / ensureQueryData / localStorage+TanStack Query / MUST NOT #1/#8 降级
- 耗时：~2 小时讨论

## P1 决策（15 个）
- 问题：token 数量 / 组件数量 / 主题初始化 / createAppRouter / forwardRef / ESLint 共存等
- 过程：分"需讨论"和"机械修复"两组。讨论项逐个过（token 36→46 / 删 createAppRouter / React 19 ref-as-prop / 去掉 ESLint 只用 Biome）
- 结论：见交接文档完整决策表
- 耗时：~1 小时

## 文件修改（两轮 agent）
- 问题：18 项决策需要落实到 10+ 个 spec 文件
- 过程：第一轮 4 agent 并行修改 → code-reviewer 发现 5 严重残留 → 第二轮 3 agent 修复 → 手工收尾 4 处
- 结论：137/0 verify 全绿 + 残留扫描清零
- 耗时：第一轮 ~15 分钟 + 审查 ~8 分钟 + 第二轮 ~5 分钟

## Codex 独立审查
- 问题：需要第三方视角做最终审查
- 过程：dispatch Codex rescue agent 做全量设计审查
- 结论：发现 4 个之前 3 轮审查都没捕获的问题（路由树 code 唯一性 / L3 API 漂移 / RouterContext 类型 / i18n 双 augmentation）。3 个已确定方向，1 个待讨论。**均未落实到文件**
- 耗时：~30 分钟
