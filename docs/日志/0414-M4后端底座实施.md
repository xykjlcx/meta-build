## Phase 0：基础设施准备（5 commits）

- 问题：项目无 Lombok，Lombok 1.18.34 不兼容 JDK 25
- 过程：添加 Lombok 后发现 `ExceptionInInitializerError: TypeTag :: UNKNOWN`，查到 JDK 25 的 `com.sun.tools.javac` 内部 API 变化导致
- 结论：升级 Lombok 1.18.34→1.18.44 + 在 root pom 显式配置 `annotationProcessorPaths`
- 其他：ArchUnit ModuleBoundaryRule 改 slices API、MdcTaskDecorator、SharedRedisContainer

## Phase 1：infra 基础设施层（12 commits）

7 个 infra 模块从占位到完整实现：i18n / exception / security / cache / rate-limit / captcha + jooq 补全 + observability 补全。

### Review 发现 3 Critical
1. **JWT Stateless → Simple**：SaTokenJwtConfig 用了 `StpLogicJwtForStateless`（无 Redis session），导致 `StpUtil.getSession()` 每次返回空会话。改为 `StpLogicJwtForSimple`
2. **LocalDateTime → OffsetDateTime**：AuditFieldsRecordListener 用 `LocalDateTime.now()` 但 DDL 是 `TIMESTAMPTZ`（对应 OffsetDateTime）
3. **TraceIdFilter userId 时序错误**：userId 在 filterChain 完成后才写入 MDC，改为独立的 `UserIdMdcInterceptor`

### 新增规则候选
- **Lombok JDK 兼容性**：Lombok 版本必须与实际运行的 JDK 版本兼容（不是 target version）。JDK 25 需要 Lombok 1.18.44+，JDK 21 用 1.18.34 即可。每次升级 JDK 需验证 Lombok 兼容性。（已记录在全局 rules）
- **mvn install 再跑 ArchUnit**：infra-archunit 新增规则后，必须先 `mvn install -DskipTests` 将新 jar 安装到本地仓库，否则 ArchitectureTest 引用的是旧 jar 中的旧字段名，报 NoSuchFieldError
