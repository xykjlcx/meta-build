# ADR 必须先于代码/文档变更

## 规则
任何翻转既有决策、引入新架构概念的行为，**先写 ADR 再改代码/specs**。不允许"先改了再补 ADR"。

## Why
2026-04-12 后端大修订中，ADR-0009~0012 对应的决策（表前缀 sys→mb、审计日志改操作日志、version 按需、全局时间策略）先改了 specs 但没写 ADR，遗留到次日补写。日志明确记录"翻转决策先写 ADR 是铁律，本会话违反了"。先改后补会导致中间状态文档无决策追溯，其他 agent 不知道为什么改了。

## How to apply
- 发现需要翻转决策时，第一个动作是在 `docs/adr/` 创建新 ADR 文件
- ADR 写完后再改 specs/代码
- 批量翻转多个决策时，可以并行写多个 ADR，但必须在改代码前全部完成
- CLAUDE.md 的 ADR 索引表同步更新
- **参考**：ADR 目录 `docs/adr/`，编号格式 `0001-简短描述.md`（四位数字递增，当前最大 0012）
